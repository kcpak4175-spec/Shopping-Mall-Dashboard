# TDD (Test-Driven Development) 프롬프트 가이드

이 프로젝트에서 새로운 기능을 개발하거나 수정할 때 AI 어시스턴트에게 TDD 방식으로 작업을 요청하기 위한 프롬프트 템플릿입니다. 효율적이고 안정적인 코드 품질을 위해 아래 프롬프트를 복사하여 사용하세요.

---

## 📋 TDD 요청 프롬프트 템플릿

```text
다음 기능 요구사항을 바탕으로 TDD(Test-Driven Development) 원칙을 엄격하게 지켜서 기능을 구현해줘.

[기능 요구사항]
(여기에 구현할 기능이나 수정할 내용을 상세히 적어주세요.)

[필수 준수 규칙]
1. 순환 주기 엄수 (Red, Green, Refactor)
   - Red: 실제 구현 코드를 작성하기 전에 '실패하는 테스트 코드'를 먼저 작성해줘.
   - Green: 방금 작성한 테스트를 통과할 수 있는 '최소한의 프로덕션 코드'를 작성해줘.
   - Refactor: 테스트가 통과하면, 중복을 제거하고 가독성을 높이는 등 코드를 리팩토링해줘. 이 과정에서도 테스트는 항상 통과해야 해.

2. 테스트 종류별 모두 작성 (Unit, Integration, E2E)
   - Unit Test: 개별 함수, UI 컴포넌트(단일 책임), 훅(Hook) 등의 단위 테스트를 작성해줘.
   - Integration Test: 여러 컴포넌트의 결합, 유스케이스와 Repository(인프라) 등 모듈 간의 통합 테스트를 작성해줘.
   - E2E Test: 사용자의 실제 브라우저 사용 시나리오를 가정한 E2E(End-to-End) 테스트를 작성해줘.

3. 커버리지 확보
   - 테스트 커버리지(Coverage)는 반드시 80% 이상이 되도록 유지하며 테스트 케이스를 촘촘히 작성해줘.

4. 최종 검증 및 트러블슈팅
   - 모든 구현과 리팩토링이 끝난 후, 전체 테스트 스위트를 다시 실행해서 모든 테스트가 정상적으로 통과하는지 완벽하게 확인해줘.
   - 만약 단 하나의 테스트라도 통과하지 못하면, 왜 통과하지 못했는지 그 원인을 자세히 분석해서 설명하고, 코드를 수정하여 최종적으로 100% 통과하도록 만들어줘.
```

---

## 💡 프롬프트 사용 팁
- 이 프롬프트를 사용할 때는 현재 프로젝트 환경(Next.js, Jest, React Testing Library, Playwright/Cypress 등)에 맞춰 테스트 프레임워크를 명시적으로 언급해주면 더욱 정확한 코드를 얻을 수 있습니다.
- "기능 요구사항" 부분에는 `features` 폴더의 어떤 도메인에 추가될 기능인지 구체적으로 명시하는 것이 좋습니다.
